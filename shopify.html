Shopify Data Science Intern Challenge

Fall 2022 — Answering real analytical questions with Python and SQL

← Back to Portfolio

This project is my submission to the Shopify Data Science Intern Challenge for Fall 2022. The challenge was split into two parts. The first used Python to investigate a statistical problem with average order value calculations. The second used SQL to answer specific business questions about a retail dataset. It was a good test of both analytical thinking and technical execution.

Tools used: Python (Pandas, Matplotlib, Seaborn), SQL, Statistical Analysis

Part 1: The Average Order Value Problem

Shopify provided data from 100 sneaker shops, each selling one model of shoe. The naive calculation of average order value (AOV) came out to $3,145.13. Given that these are sneaker shops selling a relatively affordable product, something was clearly off. The challenge was to figure out what was wrong and suggest a better metric.

What Was Going Wrong

After exploring the data in Python, I found two things that were skewing the AOV calculation:

  • At least one store was selling sneakers for $25,725 per unit, far above the typical price range. This pulled the mean up significantly
  • Two specific stores, shop IDs 42 and 78, had order amounts that were dramatically higher than every other store. A scatter plot of order amounts by shop ID made this immediately visible

I created a calculated column for price per unit by dividing order amount by total items. The average price per unit came out to $387, but the range was $90 to $25,725. That range tells the real story.

The Better Metric

Removing outliers was considered but rejected because it would exclude shops 42 and 78 entirely, which represent real business activity. Instead, the median is the better measure of central tendency here because it is not influenced by extreme values.

Answer: The median order value is $284.00, which is a far more accurate representation of a typical order than the $3,145.13 mean.

Part 2: SQL Questions

The second part of the challenge used a retail database to answer three business questions using SQL joins and aggregations.

a) How many orders were shipped by Speedy Express?

SELECT ShipperName, COUNT(O.ShipperID) AS Orders_Shipped
FROM Orders o
INNER JOIN Shippers s ON o.ShipperID = s.ShipperID
WHERE ShipperName = 'Speedy Express'
GROUP BY o.ShipperID

Answer: Speedy Express shipped 54 orders in total.

b) What is the last name of the employee with the most orders?

SELECT o.EmployeeID, e.LastName, COUNT(o.OrderID) AS Total_Orders
FROM Orders o
INNER JOIN Employees e ON o.EmployeeID = e.EmployeeID
GROUP BY o.EmployeeID
ORDER BY Total_Orders DESC
LIMIT 1

Answer: The employee with the most orders is Peacock with 40 orders.

c) What product was ordered the most by customers in Germany?

SELECT SUM(od.Quantity) AS Total_Orders, p.ProductName
FROM OrderDetails od
LEFT JOIN Products p ON p.ProductID = od.ProductID
LEFT JOIN Orders o ON o.OrderID = od.OrderID
LEFT JOIN Customers c ON c.CustomerID = o.CustomerID
WHERE c.Country = 'Germany'
GROUP BY p.ProductName
ORDER BY Total_Orders DESC
LIMIT 1

Answer: The most ordered product by customers in Germany is Boston Crab Meat.
